# SBM-MANAGER вҖ” DocumentaciГіn TГ©cnica

## DescripciГіn General

SBM-MANAGER es una aplicaciГіn de administraciГіn de franquicias desarrollada con **Vue 3** (Composition API) para el frontend y **Django REST Framework** para el backend, ambos orquestados con **Docker**. El sistema permite la gestiГіn CRUD de franquicias y sus estados, con autenticaciГіn bГЎsica, paginaciГіn, bГәsqueda y una interfaz inspirada en Django Admin Suit.

## Docker y Variables de Entorno

- **Frontend y backend corren en contenedores separados.**
- Variables de entorno para la API y autenticaciГіn se inyectan vГӯa Docker Compose y `.env`.
- Ejemplo de variables relevantes:
  - `VUE_APP_API_URL=http://localhost:8000/api/`
  - `VUE_APP_BASIC_AUTH_USER=usuario`
  - `VUE_APP_BASIC_AUTH_PASS=contraseГұa`

## Frontend (Vue 3)

### Componentes Principales

- **CRUDGridComponent.vue**: Tabla dinГЎmica con paginaciГіn, bГәsqueda, selecciГіn mГәltiple, botones de acciГіn (Configurar, Eliminar), integraciГіn con la API y estilos tipo Admin Suit.
- **Formularios de Franquicia**: Permiten crear y editar franquicias, con validaciГіn y feedback visual.
- **Sidebar**: NavegaciГіn lateral con enlaces e iconos, inspirado en Django Admin Suit.
- **Vue Router**: Configurado para navegaciГіn entre Dashboard y Franquicias.

### Vue Router

- Definido en `src/router/index.js`.
- Rutas principales:
  - `/dashboard` вҖ” Vista principal
  - `/franquicias` вҖ” AdministraciГіn de franquicias

### AutenticaciГіn y Axios

- **AutenticaciГіn Basic Auth**: Centralizada en `api/axios.js`, usando variables de entorno.
- **InyecciГіn de variables**: Usando `process.env` y Docker Compose.
- **Interceptors**: Manejo de errores y autenticaciГіn en todas las peticiones.

---

## Backend (Django REST)

### Endpoints Principales

- `GET /api/franchises/` вҖ” Listado de franquicias (soporta paginaciГіn y bГәsqueda)
- `POST /api/franchises/` вҖ” Crear franquicia
- `PUT /api/franchises/<id>/` вҖ” Editar franquicia
- `POST /api/franchises/soft_delete/` вҖ” Soft delete (cambia estado a 2)
- `GET /api/franchise_states/` вҖ” Listado de estados de franquicia

#### Ejemplo de Endpoint de BГәsqueda y PaginaciГіn

```
GET /api/franchises/?search=texto&page=1&page_size=20
```
Respuesta:
```json
{
  "count": 42,
  "next": "http://localhost:8000/api/franchises/?page=2&page_size=20",
  "previous": null,
  "results": [
    {
      "id": 1,
      "name": "Franquicia 1",
      "siglas": "F1",
      "state": 1,
      "field_verbose_names": {"name": "Nombre", "siglas": "Siglas", "state": "Estado"}
    },
    ...
  ]
}
```

#### Soft Delete
- `POST /api/franchises/soft_delete/` con body `{ "ids": [1] }` cambia el estado de la franquicia a 2 (eliminado).

## Flujo de CRUD y BГәsqueda

1. **Listado**: El componente `CRUDGridComponent` carga la lista paginada desde la API.
2. **BГәsqueda**: El input de bГәsqueda hace peticiones al endpoint con el parГЎmetro `search`.
3. **PaginaciГіn**: NavegaciГіn entre pГЎginas usando los parГЎmetros `page` y `page_size`.
4. **Crear/Editar**: Formularios modales permiten alta y ediciГіn, usando POST/PUT.
5. **Soft Delete**: El botГіn Eliminar realiza un POST a `/soft_delete/`.
6. **SelecciГіn mГәltiple**: Checkbox para seleccionar filas, con contador y acciones masivas.

## Consideraciones de Desarrollo y Despliegue

- **Variables de entorno**: Usar siempre variables para URLs y credenciales.
- **CORS**: Configurar correctamente en Django para permitir acceso desde el frontend.
- **Docker**: Usar `docker-compose up` para levantar todo el stack.
- **Estilos**: Inspirados en Django Admin Suit, usando Bootstrap y clases personalizadas.
- **ValidaciГіn**: Formularios con validaciГіn y feedback visual.
- **Escalabilidad**: Se recomienda mantener la paginaciГіn en el backend para eficiencia.
